緊急要請

親愛なる仲間諸君へ

1967年、ペンシルバニア州モンローヴィレにて、新しいショッピングセンターの建設中、 作業員たちは太古のスクロールの貯蔵庫を含むヴォールトの存在を掘り当てた。 そのほとんどは修復不可能なほどダメージを受けていたが、紀元前200年付近の、 これまで長らく疑問視されていた神秘的社会の存在を裏付けることになったのは間違いない。

発見された文書の翻訳を基にすると、 かつて『束縛変数教』と呼ばれる社会があったということはどうやら間違いなさそうだった。 デジタルコンピュータの出現まで、およそ2000年も前に、彼らは計算に対して非常に熱心に取り組んでいたようだ。

モンローヴィレで発見されたスクロールが砂岩のコンピューティングマシン用のものだとわかるにつれて、 ほとんどの研究者たちはこれを詩的に表現しただけのメタファであり、 「コンピュータ」というのも実際には人間自身のことであって、 羊皮紙にリードペンで計算を描く怖ろしく退屈な作業であったのだと決め付けた。 一部では、都市レベルの、壮大に降り注ぐ砂のマシンだったのではないかと推測してはいたが、 そういった装置の物的証拠は何も見付かっていなかった。

モンローヴィレに集積された文書のうち、無傷のものは非常に長い古写本であり、 誰も知らない言語で、人間には不可能な正確さで書かれていた。 それはこの宗教の研究の傑作であるとして、また、 歴史を紐解く新たなる一歩そのものとして受け入れられた—そして、 おそらくは計算技術の向上のためにも。 残念ながら古写本の解析は難しく、そのまま10年を過ぎたのち、 モンローヴィレ文書の探求はいつしか闇へと葬られた。 1978年からその古写本は、カーネギー自然博物館の地下へ保管されることとなった。

2週間前、CMUのコンピュータ科学関係の建物を発掘しようと現場に来ていた作業員たちは、 モンローヴィレの古写本の解析のロゼッタストーン（†キー）となるであろう、 刻印の施されたタブレットセットを発見した。 タブレットは明らかに例の宗教のコンピューティングデバイス "Universal Macine" を指し示していた。未だいずれのマシンに使えるのか判別できないものもあるにはあったが、 合理的に考えれば、モダンハードウェアでエミュレートするというのはシンプルな課題だった。

現時点で間違いなく言えることは、この古写本は Universal Machine 上で実行されるように設計されたものであるということだ。 古写本を一通り見た限り、古写本はどうやら洗練されたプログラミングアイディアを採用しているらしい。 例えそれが、我々の目には多少なりとも奇妙に見えたとしてもだ。 例の宗教が最新のエレクトロニクスや型理論にまで到達していたかも知れない可能性を考えると、 文化とは何と不思議なものだろうという思いを拭い去ることができない。

私は古写本の研究会を開くための場所を提供してもらおうと、 CMUプログラミンググループの筆頭者たちに助けを求めた。 この研究会に、ぜひ君も参加してもらえないだろうか。 古写本と Universal Machine (UM) の仕様の翻訳は、 このウェブサイトのダウンロードから手に入れられる。 君が UM を実装し、君自身が古写本の探索を始めてくれることを期待している。 暗号解除キーを入力するプロンプトが表示されたら、以下の文字列を入力してほしい。 (\b.bb)(\v.vv)06FHPVboundvarHRAk

文化の学術的な出版物は我々の興味深い項目でもある。 というのは、例の宗教の雑誌が砂岩のタブレットを循環しているからであり、 編集者は非常に限られた文字数を強いられているからだ。 結果として出版側は積極的に記事を圧縮していかなければならない。 その独特の形式は以下のようになる。

PUZZL.TSK=100@1001|14370747643c6d2db0a40ecb4b0bb65

出版物は常に変動する値だ。中には他のものよりも価値の高い表現をされるものもある。 この宗教の発刊プロセスを把握さえできていれば、 古写本にメカニズムがあると仮定し、発刊周期の確認と合計値を計算することができるのだが。

私個人としては、この宗教の研究方式に大いに感銘を受け、 たとえ孤独な人生を送ることになっても、 残る日々をすべてを計算とプログラミングに費すことを心に決めた。 だが、そんな修道士に転職する前に、 世界が『古写本』が解読する瞬間をぜひとも見てみたいのだ。

だからこそ頼みたい。私がすべてを捨て去る時、 2006年7月24日(EDT)までにできるだけ多くの出版物を送ってほしいのだ。 私の仲間である CMU POP グループのメンバー達は、きっとその時間までには、 努力を惜しまずに最高の貢献をしてくれたチームの面々が確信をもって、 特別な、ハッとさせられるような結果をもたらしてくれると保証してくれた。

力を貸してほしい。そして協力に感謝を。

Reference Implementation

um.um (1024 bytes)
CMU Reference implementation of the Universal Machine.

This implementation supports all UM programs, including uncompressed .um files and self-decompressing .umz files. To use this implementation to run a UM binary called c.um, simply concatenate the two files together:

cat um.um c.um > cmu.um

The resulting binary can be run in any compliant universal machine implementation, including itself.
